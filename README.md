# Campfire Cockpit v6.6

多模型对比分析引擎。多个 AI 独立思考同一个问题，系统找出分歧点，量化一致度，生成综合结论。

## 为什么用这个

单模型回答有盲区——它不会告诉你自己哪里可能错了。Campfire 让不同模型各自给出判断，自动对比差异、量化一致度，再由第三个模型综合出结论。你能看到哪些观点是共识，哪些是真正的争议点。

适合：需要高可信度的决策、开放性问题探索、验证 AI 回答的可靠性、深度研究与分析。

## 快速开始

1. 浏览器打开 `campfire.html`（推荐 Chrome/Edge）
2. 点右上角 `⚙ 设置`，填写 API 地址和 Key，点 `⚡ 测试` 确认连接
3. 关闭设置面板（自动保存）
4. 输入问题，按 **Enter** 开始

默认模型（可在设置中下拉选择或手动输入）：

| 角色 | 默认模型 | 说明 |
|------|----------|------|
| Mind 1 (α) | claude-opus-4-6 | 分析者 A |
| Mind 2 (β) | gpt-5.4 | 分析者 B |
| Scan | minimax-m2.5-private | 差异分析 |
| Synthesis | gemini-3.1-pro-preview | 综合结论 |
| Search | gpt-5.4 | 联网搜索（可选） |

## 两种模式

### 篝火模式（默认）

```
[联网搜索] → Mind α + Mind β 各自分析 → Scan 找出分歧 → 分歧大则再来一轮 → 综合结论
```

- 两个模型用相同提示词、不同底层模型，确保思路多样性
- Scan 量化一致度：≥70% 自动停止迭代，<70% 继续下一轮
- 问题模糊时 Scan 会主动暂停，要求你补充具体信息
- 可设置 1/2/3/5 轮，每个会话独立配置

### 极简模式

单模型直接回答，不做对比。用顶部开关切换。

## 界面说明

```
┌─ 顶栏 ─────────────────────────────────────────────┐
│ [☰档案库] [➕New]  🔥Campfire  [状态]  [模式] [轮数] [⚙设置] │
├────────┬─────────────────┬──┬──────────────┤
│ 标签栏  │                 │  │              │
│ 标题    │  左侧：执行过程   │拖│ 右侧：综合结论  │
│ 状态    │  (双模型输出)    │动│ (可钉选摘录)   │
│ 计时    │                 │条│              │
│        │  [输入框] [IGNITE]│  │ [导出][导出全部] │
│        │  [🌐联网]         │  │              │
└────────┴─────────────────┴──┴──────────────┘
```

**标签栏**：宽标签显示会话标题、状态图标、计时器。支持最多 3 个会话并行。点击切换，双击重命名。

**左侧（执行日志）**：实时流式显示每轮 Mind 输出和差异分析。悬停出现 `⧉` 复制和 `⊞` 钉选按钮。

**右侧（结论面板）**：综合结论（生成中显示加载动画，完成后出现操作按钮）。`🔄 重新生成` 可换模型重跑结论，`✏️ 编辑重跑` 修改原始问题重新分析，`↩ 继续追问` 接续对话。

## 运行中交互

- **停止**：点击"停止"按钮，已生成内容完整保留，不丢失
- **追加输入**：会话运行中可继续输入，自动排队，当前分析完成后无缝衔接为追问
- **编辑重跑**：点击右侧面板问题旁的 `✏️ 编辑重跑`，修改后 Enter 重新执行

## 跨对话记忆

系统自动建立历史会话索引，新对话时参考相关的过往结论。

1. 会话完成保存后，自动提取关键词建立索引（存入档案库 `memory.json`）
2. 新对话开始时，关键词匹配相关历史
3. 命中的完整综合结论（最多 2 条）注入 Mind 的上下文

注入的是原始结论全文，不是压缩摘要。提示词标注"仅供参考"，不会盲从历史。

**迁移工具**：现有历史会话可用 `memory-indexer.html` 一次性批量建立索引。

## 断点续传

网络中断时，系统保存当前执行进度，显示 `🔄 断点续传` 按钮，一键从中断步骤继续。

## 联网搜索

1. 点击输入框下方 `🌐 联网` 启用（变为青色）
2. 篝火模式下 α 和 β 各自独立搜索；极简模式单次搜索
3. 搜索结果自动注入分析上下文

## 档案库

会话、配置、记忆索引全部存在本地文件夹，结构清晰：

```
vault/
├── config.json       ← API 配置 + 提示词
├── memory.json       ← 跨对话记忆索引
└── sessions/
    └── Campfire_*.json  ← 会话记录
```

**首次使用**：`⚙ 设置` → `📂 挂载目录` → 选择本地文件夹 → 授权。之后自动保存。

**恢复历史**：`☰ 档案库` → 点击记录 → 加载为新标签 → 可继续追问。

**迁移**：拷贝整个文件夹到另一台电脑，挂载即可恢复全部数据。

## 注意事项

- **浏览器**：Chrome 或 Edge（需 File System Access API）
- **API**：OpenAI 兼容 chat/completions 接口，支持 SSE 流式输出
- **数据安全**：全部本地存储，不经过第三方服务器
- **模型搭配**：Mind 1 和 Mind 2 选输出深度相近的模型（如 Claude + GPT），避免差异分析失真
- **轮数**：大多数问题 2-3 轮足够，5 轮用于深度争议性话题
